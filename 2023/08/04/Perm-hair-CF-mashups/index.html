CF 1714E. Add Modulo 10

The problem is basically as follows:

For each test case, you are given an array of nnn (n≤2⋅105n \leq 2\cdot10^5n≤2⋅105) integers,
you can apply an operation ai=ai+(ai mod 10)a_i = a_i + (a_i \space \text{mod} \space 10)ai​=ai​+(ai​ mod 10) on any elemment,
Is it possible to make every element in the array the same by using a finite ammount of operations?

An Easy math pattern finding problem.
We can try by applying the operation on all unit digits and observe the pattern of which the unit digit changes.

Example:
9 →\rarr→ 8 →\rarr→ 6 →\rarr→ 2 →\rarr→ 4 →\rarr→ 8 →\rarr→ 6 →\rarr→ 2…
7 →\rarr→ 4 →\rarr→ 8 →\rarr→ 6 →\rarr→ 2…

After trying all of them, its easy to notice that there are two patterns:
If the unit digit is 0 or 5. After one operation the unit digit will become 0.
Else, after some ammount of operation, the unit digit will land on a 8 -> 6 -> 2 -> 4 infinite cycle.

Now we can obtain the solution:
If there is any element with 0 or 5, we apply the operation to every element once and check if they are all the same.
Else, we apply operations until they all have the same unit digit, and check if they have the same value modulo 20.
The reason its 20 is because one cycle is 8 -> 6 -> 2 -> 4, which adds up to 20.

Code (flg1 is to check if there is a element whose unit digit is 5 or 0, flg2 is check if its possible)

Time Complexity: O(n)O(n)O(n)

CF 1249C2. Good Numbers (hard version)

The problem is basically as follows:

A good number is one that can be made by using distinct powers of 3.
For each test case, you are given a number nnn (n≤1018n \leq 10^{18}n≤1018), output the smallest good number greater or equal to nnn.

Honestly not sure why this question is rated 1500…

The idea is really simple: Maintain an array preprepre, where prei=3i+3i−1+...+30pre_i = 3^i + 3^{i-1} + ... + 3^0prei​=3i+3i−1+...+30.
You want to find the one that is greater or equal to nnn, and from large to small greedily remove powers of 3 if possible.

CF 1081B. Farewell Party

The problem is basically as follows:

Given an array, each iii belongs to some group. aia_iai​ denotes how many people are in different groups then iii, check if theres a valid grouping that can satisfy the array’s constraints.

First, lets think about what aia_iai​ actually means:

aia_iai​ denotes how many people are in different groups then iii.

Can be changed to:

n−ain - a_in−ai​ denotes how many people are in the same group as iii.

Although it doesn’t seem much, we can actually use this knowledge to our advantage:
This means that for a possible combination, there must be n−ain - a_in−ai​ people whose value in the array is aia_iai​!
Further more, there might be multiple groups with the same amount of people in it.
So we can change our previous conclusion to there must be a multiple of n−ain - a_in−ai​ people whose value in the array is aia_iai​!
After knowing this, we just need to assign groups to each person and AC this question :D

Time Complexity: O(n)O(n)O(n)

CF 1543D1. RPD and Rap Sheet (Easy Version)

The problem is basically as follows:

There is a password xxx between 000 to n−1n - 1n−1 (n≤2⋅105n \leq 2\cdot10^{5}n≤2⋅105). You can guess the password qqq (q≤nq \leq nq≤n) times.
After each failed guess yyy, the password will become zzz where x⊕z=yx⊕z = yx⊕z=y (⊕⊕⊕ denotes binary XOR)

The problem statement actually gave me a big hint:

Password between 000 to n−1n - 1n−1 and guess the password qqq (q≤nq \leq nq≤n) times.
→\rarr→ Try all passwords from 0 to n - 1!

Now, the problem is that the password changes after each failed attempt. How would we solve this?
Lets observe the encoding function of the password:

x⊕z=yx⊕z = yx⊕z=y →\rarr→ x⊕x⊕z=x⊕yx⊕x⊕z = x⊕yx⊕x⊕z=x⊕y →\rarr→ x⊕y=zx⊕y = zx⊕y=z (Because x⊕x=0x⊕x = 0x⊕x=0)

Denote the password for step iii as xix_ixi​, for a failed query qiq_iqi​, the password becomes xi⊕qix_{i}⊕q_ixi​⊕qi​ = x⊕qi⊕qi−1...⊕q1x⊕q_i⊕q_{i-1}...⊕q_1x⊕qi​⊕qi−1​...⊕q1​ !
Thus if we also encode our query qiq_iqi​ with ⊕qi−1...⊕q1⊕q_{i-1}...⊕q_1⊕qi−1​...⊕q1​, the encoded value for xix_ixi​ is the same as the original xxx to query qiq_iqi​!
