pB was a really tricky one. The problem gives you an array of integers, and you have an operation ai=ai+1a_i = a_i + 1ai​=ai​+1. It asks the minimum number of operations that makes the product of the array divisible by kkk (1≤k≤5(1 \leq k \leq 5(1≤k≤5).

The first idea is we can make the array [a0,a1,a2...an−1][a_0, a_1, a_2 ... a_{n - 1}][a0​,a1​,a2​...an−1​] into [a0%k,a1%k,...,an−1%k][a_0 \% k, a_1 \% k ,..., a_{n - 1} \% k][a0​%k,a1​%k,...,an−1​%k],
because of the fact that (a⋅b)%k=((a%k)⋅(b%k))%k(a \cdot b) \% k = ((a \% k) \cdot (b \% k)) \% k(a⋅b)%k=((a%k)⋅(b%k))%k.
Obviously, if there is a 000 in the array, the answer is 000.
Else, we want to try to find the number that is closest to kkk (so we can do the operation until it is k).
But this does not work in all cases, for example, [1,1],k=4[1, 1], k = 4[1,1],k=4, the answer should be 222 (add 1 on both elements).
I was stuck we for a while, but notice the constraint for kkk! It turns out, the case I mentioned would only occur for 444! for 2,3,52, 3, 52,3,5, we can just check it as usual, while we need to also check the way to make 2 2’s when k=4k = 4k=4.

pC was extremely easy for me. I solved it in under 7 minutes!
The problem is: Given an array, calculate the number of subarrays such that it occurs in the array as a subsequence exactly once.

Lets say if we have a subarray a[l,r]a[l, r]a[l,r], if this subarray occurs in the array as a subsequence more than once, this implies that there exists (at least) one r′>rr' > rr′>r s.t a[r′]=a[r]a[r'] = a[r]a[r′]=a[r], or (at least) one l′<rl' < rl′<r s.t a[l′]=a[l]a[l'] = a[l]a[l′]=a[l], or even both.
So, we can find candidates for left/right end point by checking if there isn’t the same element on its left/right.

Example:

111 777 777 222 333 444 333 222 111 -> the array
111 111 000 111 111 111 000 000 000 -> potential left endpoint candidates
000 000 111 000 000 111 111 111 111 -> potential right endpoint candidates

Then the answer is just the sum of for each end point, the number of right candidates on its right.
We can calculate this quickly by using an prefix sum.
