CF 1904D. Set To Max (Hard Version)

There are two versions of the problem, with the difference being n≤1e3n \leq 1e3n≤1e3 on easy, and n≤2e5n \leq 2e5n≤2e5 on hard.

First of all, I imagined the operation as choosing an index iii and letting a[i]a[i]a[i] as the max of a operation, and we can “extend” a[i]a[i]a[i] to left and right.

The only restriction when we can’t keep extending further a[i]a[i]a[i] is when we are met with a larger a[j]a[j]a[j], because $a[i] wouldn’t be the biggest anymore if the range we chose consists a[j]a[j]a[j].

So basically, for an index iii, we can extend left and right to index jjj and kkk, where j<i<kj < i < kj<i<k, a[j]>a[i]a[j] > a[i]a[j]>a[i] and a[k]>a[i]a[k] > a[i]a[k]>a[i].

Now, we wouldn’t always want to extend as large as we can, as if theres a index jjj in it with b[j]<a[i]b[j] < a[i]b[j]<a[i], we would obviously fail making both arrays equal as we can only make it bigger not smaller.

So now we have two constraint as boundaries:

On the left side find jjj, where j<ij < ij<i, a[j]>a[i]a[j] > a[i]a[j]>a[i] or b[j]<a[i]b[j] < a[i]b[j]<a[i],
On the right side find kkk, where k>ik > ik>i, a[k]>a[i]a[k] > a[i]a[k]>a[i] or b[k]<a[i]b[k] < a[i]b[k]<a[i].

And for the boundaries we choose the largest jjj and smallest kkk.

Now we have a boundaries (ji,ki)(j_i, k_i)(ji​,ki​) for each index, where we can set a[ji]a[j_i]a[ji​] to a[ki]a[k_i]a[ki​] to a[i]a[i]a[i],and that we are sure that this operation will only make the array closer to what we want.

Its then sufficient to apply all the operations from the smallest a[i]a[i]a[i] to the biggest and checking if the array is the same.

The easy version is just manually doing the operations everytime with another for loop, taking O(n2)O(n^2)O(n2) time.

The hard version requires us to do all operations quicker than O(nlogn)O(nlogn)O(nlogn).

Lets say that we already have all the boundaries for each index and we need to apply it, then we would need a range set element data structure to help us do this, which lazy tag segment trees can do easily in O(nlogn)O(nlogn)O(nlogn).

The problem now becomes finding (ji,ki)(j_i, k_i)(ji​,ki​) for all iii quickly, which actually can we done with segment trees again!

There are two cases for each bound, for left bound, either b[j]<a[i]b[j] < a[i]b[j]<a[i] case or a[j]>a[i]a[j] > a[i]a[j]>a[i], and we want to find the biggest one.

I will talk about how to find the max jjj such that a[j]>a[i]a[j] > a[i]a[j]>a[i], as others are pretty much the same.

For this, we would want a point set/add range max segment tree, where the index is the value and inside stores the largest index with that value.

We traverse from left to right, each time we find the max value between a[i]+1a[i] + 1a[i]+1 to the max value, this way we can get the biggest index jjj that a[j]>a[i]a[j] > a[i]a[j]>a[i], and we just set index iii to a[i]a[i]a[i] in the segment tree.

Now just do this four times to find the bound, and do the updates with range set segment tree.

Time complexity: O(nlogn)O(nlogn)O(nlogn)
